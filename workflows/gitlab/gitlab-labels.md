# GitLab 标签体系

## 标签分组

### 类型（scoped label: `类型::`）

| 标签 | 颜色 | 用途 |
|------|------|------|
| `类型::feat` | #28a745 | 新功能 |
| `类型::fix` | #d73a4a | Bug 修复 |
| `类型::refactor` | #0075ca | 重构 |
| `类型::docs` | #7057ff | 文档 |
| `类型::test` | #008672 | 测试相关 |
| `类型::chore` | #cfd3d7 | 杂项 |

### 优先级（scoped: `优先级::`）

| 标签 | 颜色 | 用途 |
|------|------|------|
| `优先级::P0` | #b60205 | 阻塞级 |
| `优先级::P1` | #d93f0b | 严重 |
| `优先级::P2` | #fbca04 | 一般 |
| `优先级::P3` | #c5def5 | 轻微 |

### 状态（scoped: `状态::`）

| 标签 | 颜色 | 用途 |
|------|------|------|
| `状态::待评审` | #e4e669 | 需求/用例/提测待评审 |
| `状态::开发中` | #1d76db | 开发进行中 |
| `状态::待测试` | #0e8a16 | 已提测 |
| `状态::测试中` | #5319e7 | 测试进行中 |
| `状态::待修复` | #d73a4a | Bug 待开发处理 |
| `状态::修复中` | #f9d0c4 | 开发修复中 |
| `状态::待回归` | #fef2c0 | 修复完待回归 |
| `状态::已关闭` | #c5def5 | 已完成 |
| `状态::挂起` | #cfd3d7 | 暂不处理 |

### 版本（scoped: `版本::`）

| 标签 | 用途 |
|------|------|
| `版本::v1.2.3` | 关联具体版本 |

### 模块（scoped: `模块::`）

| 标签 | 用途 |
|------|------|
| `模块::user` | 用户中心 |
| `模块::order` | 订单 |
| `模块::payment` | 支付 |
| ... | 按实际模块定义 |

### 特殊标签

| 标签 | 颜色 | 用途 |
|------|------|------|
| `Bug` | #d73a4a | Bug |
| `提测` | #0e8a16 | 提测单 |
| `回归` | #fbca04 | 回归测试 |
| `线上问题` | #b60205 | 线上故障 |
| `技术债` | #f9d0c4 | 技术债务 |

## 使用约定

1. **每个 Issue/MR 必须至少有**：类型、优先级、状态
2. **状态必须是唯一的**（scoped label 机制会自动替换）
3. **重大 Bug 必须加模块标签**
4. **线上问题单独跟踪**，便于做故障复盘

## 创建方式

在 GitLab 项目 Settings → Labels 中批量创建，或通过 API：

```bash
# 示例：通过 GitLab API 批量创建标签
curl --request POST \
  --header "PRIVATE-TOKEN: $GITLAB_TOKEN" \
  --data "name=类型::feat&color=#28a745" \
  "https://gitlab.example.com/api/v4/projects/$PROJECT_ID/labels"
```
