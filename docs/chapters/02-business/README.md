# 第 02 篇 · 业务篇

> **状态**: ✅ 5 个子章节完成
> **目标读者**: 产品经理 / 业务分析师（BA）/ 项目经理

## 一、本篇目标

解决业务侧的三个核心问题：
1. **需求如何规范化产出**：让下游（开发/测试）接到就能干活
2. **业务规则如何清晰表达**：避免"文档说不清，口头也说不清"
3. **AI 如何帮助业务**：用 Claude 做需求梳理、可测性分析、用户故事拆解

## 二、业务在整个研发流程中的位置

```
                    ┌─────────────────┐
                    │  业务场景        │
   市场/客户/战略 ──►│                 │──► 需求包（PRD、用户故事、验收标准）
                    │  本篇的范围      │
                    └─────────────────┘
                                        │
                                        ▼
                                    开发场景
```

### 业务输出给开发/测试的"交付包"

一个合格的业务交付包应包含：

| 交付物 | 必备 | 格式 |
|--------|------|------|
| PRD 文档 | ✅ | Markdown |
| 用户故事集 | ✅ | Markdown（Given-When-Then） |
| 业务规则表 | ✅ | Markdown 表格 |
| 业务流程图 | ✅ | Mermaid / 图片 |
| 数据字典 | 可选 | Markdown 表格 |
| 原型图 | 可选 | 链接 / 图片 |
| 竞品分析 | 可选 | Markdown |
