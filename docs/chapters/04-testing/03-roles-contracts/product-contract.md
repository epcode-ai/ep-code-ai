# 产品 → 测试 契约

## 产品交付物给测试

### 1. 需求文档（PRD）

| 项目 | 要求 |
|------|------|
| 格式 | Markdown 文件 / Confluence 页面 / GitLab Wiki（任选其一） |
| 命名规范 | `PRD-<模块>-v<版本>.md` |
| 标签 | `prd`, `sprint-<N>` |
| 必须包含 | 用户故事（As a... I want... So that...） |
| 必须包含 | 验收标准（Given... When... Then...） |
| 必须包含 | 业务规则表（条件 → 结果） |
| 必须包含 | 异常场景说明 |

### 2. 交付时间

| 阶段 | 时限 |
|------|------|
| 提供完整 PRD | 需求评审前 2 工作日 |
| 回应测试疑问 | 1 工作日内 |
| 需求变更通知 | 变更后 4 小时内 |

### 3. 需求变更处理

- 所有变更必须在需求管理系统中记录
- 重大变更（影响测试范围）必须重新评审
- 冻结期后（一般是测试开始后）原则上不接受变更

## 测试回传给产品

| 交付物 | 时机 | 格式 |
|--------|------|------|
| **需求可测性评审意见** | 评审会前 1 天 | Markdown 清单 |
| **测试策略评审邀请** | 策略完成后 | 会议邀请 + 文档 |
| **测试进度周报** | 每周五 17:00 | Markdown / 邮件 |
| **测试总结报告** | 版本结束 2 天内 | Markdown（见模板） |
| **准出报告** | 准出评审前 | Markdown（见模板） |

## 违约处理

| 场景 | 处理方式 |
|------|---------|
| PRD 缺失验收标准 | 退回产品补齐，计划延期 |
| 关键疑问 24h 未响应 | 升级至项目经理 |
| 测试开始后变更需求 | 评估影响，如需延期则项目经理决策 |
| 连续 2 个版本违约 | 项目复盘 + 流程改进 |
