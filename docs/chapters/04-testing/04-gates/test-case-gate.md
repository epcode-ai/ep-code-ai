# 用例评审门禁

## Gate In（用例评审的前置条件）

- [ ] 测试策略已批准
- [ ] 接口契约已确认
- [ ] 数据库设计已确认
- [ ] 用例已完成编写

## Gate Out（用例评审通过的标准）

### 覆盖度检查

- [ ] 核心业务流程覆盖率 100%
- [ ] 异常场景覆盖率 ≥ 80%
- [ ] 边界值覆盖完整
- [ ] 每个需求点都有对应用例

### 用例质量检查

- [ ] 步骤清晰、可执行
- [ ] 测试数据明确
- [ ] 预期结果具体（不是"应该正常"这种模糊描述）
- [ ] 优先级合理

### 评审意见

用例评审必须邀请：
- 产品：确认业务理解正确
- 开发：确认技术实现覆盖
- 测试组内：交叉评审

## 不通过的处理

| 问题 | 处理 |
|------|------|
| 覆盖度不足 | 补充用例后重评 |
| 用例不清晰 | 修订后内评 |
| 与需求理解不一致 | 与产品对齐后修订 |

## 评审产物

- 已签字的用例集
- 评审意见汇总
- 用例改动记录
