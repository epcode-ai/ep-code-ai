# 04 · 门禁标准

## 门禁的意义

> **"没有合格的输入，就没有合格的输出"**

测试守住质量的第一道防线，就是**不接收不合格的输入**。每个阶段都有明确的门禁标准，不达标即退回。

| 门禁 | 位置 | 守护者 | 详细文档 |
|------|------|-------|---------|
| 需求评审门禁 | 需求阶段 | 测试 + 开发 | [requirements-gate.md](./requirements-gate.md) |
| 用例评审门禁 | 用例设计阶段 | 测试 + 产品 + 开发 | [test-case-gate.md](./test-case-gate.md) |
| **提测达标** | 提测阶段 | **测试** | [submission-gate.md](./submission-gate.md) ⭐ |
| 准出评审门禁 | 上线前 | 测试 + PM + 运维 | [release-gate.md](./release-gate.md) |

## 门禁执行的三条铁律

1. **标准写在前面**：不能提测了才说标准是什么
2. **不达标即退回**：不迁就，不妥协，记录在案
3. **度量驱动改进**：每月统计违约情况，推动流程优化

## 门禁不是卡点，是对齐

门禁存在的真正目的：
- ❌ 不是让测试显示权威
- ❌ 不是让流程变慢
- ✅ 是让各角色**提前知道交付标准**
- ✅ 是让质量问题**左移发现**（越早发现越便宜）
- ✅ 是让测试资源**用在刀刃上**
