# 准出评审门禁

## Gate In（准出评审的前置条件）

- [ ] 全部测试阶段完成
- [ ] 回归测试通过
- [ ] 所有 Bug 已关闭或已评估可带上线
- [ ] 准出报告已编写

## Gate Out（准出评审通过的标准）

### 质量指标

- [ ] 核心用例通过率 = 100%
- [ ] 整体用例通过率 ≥ 98%
- [ ] 无 P0/P1 遗留 Bug
- [ ] Bug 收敛趋势达标
- [ ] 性能指标达标
- [ ] 安全扫描无高危问题

### 文档齐备

- [ ] 准出报告完整（见[模板](../../../../templates/testing/test-reports/)）
- [ ] 风险清单明确
- [ ] 回滚方案就绪
- [ ] 灰度验证 checklist 已提供

### 相关方确认

- [ ] 测试负责人签字
- [ ] 项目经理批准
- [ ] 运维接收确认
- [ ] 产品确认

## 准出意见三档

| 意见 | 含义 |
|------|------|
| ✅ **通过** | 可正常上线 |
| ⚠️ **有条件通过** | 带已知风险上线，有缓解措施 |
| ❌ **不通过** | 禁止上线，需修复后重新评审 |

## 不通过的处理

由项目经理召集紧急会议，决定：
- 延期上线
- 降级发布（去掉问题功能）
- 加急修复 + 二次准出评审

## 准出产物

- 已签字的准出报告（存档）
- 灰度验证 checklist（给运维）
- 监控重点项清单（给运维）
