# Flow 05 · 会话中切换供应商的影响处理

> 这是个潜在风险操作 —— 切换不当会丢上下文 / 增费用。流程要明确预警 + 给安全默认。

## 主流程

```mermaid
flowchart TD
    A[用户触发切换] -->|顶栏点击 / ⌘⇧P| B[弹切换浮层 wireframe-06]
    B --> C{用户选择}
    C -- esc / 关闭 --> Z[无操作,返回]
    C -- 选目标供应商 --> D{勾选 '在新会话中切换'?}

    D -- 是 (默认,推荐) --> E[创建新会话]
    E --> E1[新会话默认供应商 = 目标]
    E1 --> E2{当前会话有未发消息?}
    E2 -- 是 --> E3[把未发消息复制到新会话输入框]
    E2 -- 否 --> E4[新会话空状态]
    E3 --> F[侧边栏选中新会话,中栏切换]
    E4 --> F

    D -- 否 (高级) --> G[弹二次确认对话框]
    G --> G1[显示风险:<br/>- 上下文丢失<br/>- 可能费用增加<br/>- 回复偏离]
    G1 --> G2{用户输入 '切换' 确认}
    G2 -- 取消 --> Z
    G2 -- 确认 --> H[当前会话切换供应商]

    H --> H1[聊天流插入 '⚙ 已切换到 X' 系统消息]
    H1 --> H2[顶栏供应商显示更新]
    H2 --> H3[token 计数器重置 0]
    H3 --> I{下一条消息发送时}
    I --> J{新供应商是否需要重塑上下文?}
    J -- 是 (大多数情况) --> K[把历史消息以 system 消息发给新供应商]
    J -- 否 (兼容协议) --> L[直接发当前消息]
    K --> M[新供应商回复]
    L --> M
```

## "重塑上下文" 详细策略

```mermaid
flowchart TD
    A[切换后第一次发消息] --> B[计算历史消息 token 数]
    B --> C{是否超过新供应商上下文窗口?}
    C -- 否,全部塞得下 --> D[把全部历史 + 新消息一次发出]
    C -- 是,塞不下 --> E[摘要压缩:<br/>用便宜模型把早期消息压成摘要]
    E --> F[摘要 + 最近 N 条原文 + 新消息发出]
    F --> G{压缩后仍超?}
    G -- 是 --> H[弹模态: '上下文超长,需截断或新建']
    G -- 否 --> I[正常发送]
```

## 状态机

```mermaid
stateDiagram-v2
    [*] --> Idle: 会话稳定状态
    Idle --> Switching: 用户触发切换浮层
    Switching --> Idle: 取消
    Switching --> NewConversation: 选'在新会话中切换'
    Switching --> Confirming: 选'在当前会话中切换'
    Confirming --> Idle: 取消
    Confirming --> InCurrent: 输入'切换'确认
    InCurrent --> Reshaping: 下次发消息时
    Reshaping --> Idle: 重塑成功
    Reshaping --> Failed: 重塑失败 (超长 / API 错)
    Failed --> Idle: 用户决定下一步 (截断 / 新建 / 取消)
    NewConversation --> [*]: 已转移到新会话
```

## 切换的用户消息

切换的系统消息样式:

```
⚙ 12:51  · 切换供应商
   Anthropic Claude opus  →  AWS Bedrock claude-opus-4-v1

   📊 影响:
   • 历史 156k tokens 将以摘要形式重塑
   • 预估额外费用: $0.18 (摘要 + 重塑)

   [📋 看摘要预览]   [🔙 撤销切换]
```

撤销窗口: 3 分钟内可点 "🔙 撤销切换" 回到原供应商,撤销后系统消息标 "已撤销"。

第一次用户做 "在当前会话切换" 时:

```

        │  你正在做"会话内切换",这是一个高级操作。      │
        │                                               │
```

## 失败处理

切换后第一次发送失败的处理:

| 失败类型 | 行为 |
|---------|------|
| API 401/403 | 弹模态 "新供应商鉴权失败,要回退到原供应商吗?" |
| 上下文超长 | 弹模态 "无法重塑全部上下文,选择: 截断早期 50% / 新建会话 / 切回原" |
| 网络 | 重试 3 次,仍失败弹 "切换可能未生效,看右上状态" |

## 与 Sprint 1-5 的对接

- 切换日志写入会话历史 JSON,key: `provider_switches: [{ timestamp, from, to, reshape_strategy }]`
- 度量脚本 `tools/metrics/development/collect.js` 可加新维度: 周内供应商切换次数(衡量稳定性)
