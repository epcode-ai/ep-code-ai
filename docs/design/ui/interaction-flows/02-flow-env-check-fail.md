# Flow 02 · 环境检查失败的兜底引导

> 任何时机检测到环境问题(启动 / hover 状态栏 / 跑命令前)的统一处理。

## 主流程

```mermaid
flowchart TD
    A[触发环境检查] --> B[并行检查所有依赖]
    B --> C{结果分类}

    C -- 全绿 --> C1[状态栏显示 ✅]
    C -- 有 ⚠️ 但无 ❌ --> D[状态栏黄色,允许继续]
    C -- 有 ❌ --> E[状态栏红色,关键操作禁用]

    D --> D1{用户操作}
    D1 -- 忽略 --> D2[继续工作,状态栏持续黄]
    D1 -- 点 [▶ 修复] --> F[进入修复流程]

    E --> E1[禁用'发送消息'按钮]
    E --> E2[弹一次性 banner: '环境损坏,功能受限']
    E1 --> F

    F[修复流程] --> F1{依赖类型}
    F1 -- npm 包 (Claude Code 等) --> F2[内置一键安装]
    F1 -- 系统命令 (xcode-select 等) --> F3[显示命令 + 复制按钮]
    F1 -- 系统包 (apt/brew/winget 等) --> F4[显示对应包管理器命令]

    F2 --> G[打开内置终端面板,实时显示输出]
    G --> G1{安装结果}
    G1 -- 成功 --> H[自动重测,显示 ✅]
    G1 -- 失败 --> G2[显示错误日志 + '复制错误' + '重试' + '查文档']
    G2 --> F1

    F3 --> I[等待用户去外部终端跑]
    F3 --> I1[显示 '已运行?' 按钮]
    I1 -- 点击 --> J[重测]

    F4 --> I
    H --> K[关修复对话框,状态栏更新]
    J --> K
```

## 不打扰原则

- **启动时**: 不主动弹模态,只在状态栏标色,用户主动点才修复
- **跑命令前**: 如果命令依赖某个缺失项,弹模态;否则放行
- **常驻提示**: 每天最多 1 次顶部 banner,点 ✕ 关闭后当天不再提
- **修复成功**: 不打扰,只更新状态栏,2 秒淡入淡出 ✅ 提示

## 修复脚本来源

每种依赖的修复命令定义在 `tools/cross-platform/scripts/fixes/<dependency>.json`:

```json
{
  "name": "Claude Code",
  "check": "claude --version",
  "minVersion": "1.0.0",
  "install": {
    "darwin": ["npm", "install", "-g", "@anthropic-ai/claude-code"],
    "linux": ["npm", "install", "-g", "@anthropic-ai/claude-code"],
    "win32": ["npm.cmd", "install", "-g", "@anthropic-ai/claude-code"]
  },
  "manualHint": {
    "darwin": "xcode-select --install"
  }
}
```

应用启动加载这个目录,自动构建检查清单 + 修复策略。
