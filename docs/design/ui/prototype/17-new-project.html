你的项目处于什么阶段?

选对接入模式,框架才能给你最合适的引导。不确定?可以看每张卡片下面的判定条件。

A
绿地项目
全套上车 · 从 0 开始建
判定: 项目刚立项,尚未开始编码 · 团队愿意从 Day 1 走规范
适合: 新业务 / 创新项目 / 独立新系统
B
开发中项目
从现在追溯补齐
判定: 代码已开始写,但未上线 · 零散 PRD / 设计
适合: 接手半成品 · 新项目改造
C
迭代中项目
每版本加一层 · 渐进嵌入
判定: 已上线,双周/月迭代 · 有成熟但不同的流程
适合: 老项目升级 · 存量业务改造
D
稳态运维项目
聚焦运维 · 只做关键层
判定: 运行 1 年+,不再大版本迭代 · 只做 bug 修复
适合: 线上老系统 · 维护中的服务
