PRD · 离职流程 v1.2

版本: v1.2 作者: 张工(产品) 日期: 2026-04-19 ⏳ 草稿

1. 概述

1.1 背景

公司目前离职流程涉及 5 个部门(HR / 财务 / IT / 上级 / 行政),平均处理时长 7-10 工作日,员工反馈断点多。

1.2 目标

  • 端到端处理时长 ≤ 3 工作日
  • 关键步骤 透明可追溯
  • 员工 NPS ≥ 60

2. 用户与场景

2.1 角色

角色 关键动作 SLA
员工 提交申请 / 确认交接 —
直接上级 审批 / 工作交接确认 1 工作日
HR 资料审核 / 流程协调 1 工作日
IT 账号注销 / 设备回收 0.5 工作日

2.2 主流程

员工发起 → 上级审批 → HR 审核 → 并行(IT 注销 / 财务结算 / 行政回收)→ 员工确认 → 归档。

3. 功能需求

REQ-001 · 员工在线提交离职申请

背景: 当前员工需线下填纸质表格,流程不透明。

验收标准(Given-When-Then):

  • Given 员工已登录内部系统
  • When 员工进入"我的流程"→"离职申请"并填写表单(离职日期、原因、备注)
  • Then 系统创建申请单,状态 待上级审批,并发送通知给直接上级

REQ-002 · 上级审批

(省略 · 点击"源码"查看完整内容)

4. 非功能需求

  • 性能: 单个流程操作 P99 响应时间 ≤ 500ms
  • 可用性: 99.9% SLA(全年停机 ≤ 8.76h)
  • 安全: 离职相关个人信息 加密存储 + 访问审计
  • 兼容: 支持 Chrome 90+ / Safari 14+ / 企业微信内嵌
