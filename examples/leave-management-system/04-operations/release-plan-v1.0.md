## 4. 回滚方案

### 4.1 立即回滚触发条件

任一满足即回滚：
- 核心业务（发起/审批）不可用
- 错误率 > 2%
- 数据错误/丢失
- 安全漏洞暴露

### 4.2 回滚步骤

```bash
# Step 1: 关闭灰度白名单，恢复旧流程（立即生效）
kubectl -n hr set env deployment/leave-api GRAYSCALE_ENABLED=false
# 同时发群通知："请使用原邮件审批流程"

# Step 2: 应用回滚（如发生数据错乱可选）
kubectl -n hr rollout undo deployment/leave-api
kubectl -n hr rollout undo deployment/notification-worker

# Step 3: 数据回滚（仅极端情况）
psql -h hr-pg-prod -U admin -d leave -f migrations/v1.0.0/down.sql
```

### 4.3 回滚后

- 保留生产现场（日志快照、监控截图）
- 发故障报告（24h 内）
- 召开紧急复盘
