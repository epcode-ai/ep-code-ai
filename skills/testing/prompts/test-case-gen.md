# 测试用例生成

## 用途

根据需求和接口契约，批量生成测试用例，测试人员只需评审和补充业务细节。

## 使用场景

- 新需求用例设计
- 老接口补测试
- 接口变更后补充用例

## Prompt 本体

```markdown
你是一位资深测试工程师。根据以下需求和接口契约，生成结构化测试用例矩阵。

## 需求信息
[粘贴需求描述 + 验收标准]

## 接口契约
[粘贴 API 文档]

## 已有测试数据（可选）
[如有可用的测试账号、测试订单等，提供给 AI]

## 覆盖要求

### 必须覆盖
1. **正常流**（核心业务流程）
2. **参数校验**（必填、类型、长度、格式）
3. **边界值**（最小、最大、临界）
4. **异常流**（权限、资源不存在、状态冲突）
5. **安全**（SQL 注入、XSS、越权）

### 建议覆盖
6. **并发**（如涉及）
7. **幂等性**（如涉及）
8. **性能基线**（如涉及）
9. **兼容性**（如涉及多端）

## 输出格式

每条用例输出:
| 用例 ID | 模块 | 优先级 | 类型 | 前置条件 | 测试数据 | 操作步骤 | 预期结果 |

## 输出约束
- 用例 ID 格式: TC-[模块]-001
- 优先级: P0/P1/P2/P3
- 类型: 功能/接口/性能/安全/兼容
- 步骤编号清晰（1. 2. 3.）
- 预期结果具体（不是"正常"、"成功"，而是"返回 200 + code=0 + 字段 X=Y"）
- 测试数据明确（不是"正确的数据"，而是具体值）

## 附加输出

### 覆盖度评估
| 维度 | 用例数 |
|------|-------|
| 正常流 | |
| 异常流 | |
| 边界值 | |
| 安全 | |

### 需要测试数据准备
- 数据 1: [描述 + 准备方式]
- 数据 2: ...

### 需要补充的测试点（AI 识别的但没法生成具体用例的）
- 点 1: [为什么 AI 生成不了]

### 自动化适配度
对每条用例标注: 🤖 适合自动化 / 👤 建议手工 / ❓ 待评估
```

## 使用示例

**输入**: 登录接口的需求 + API 文档

**输出**: 15 条用例，覆盖正常、异常、边界、安全
