# 测试报告生成

## 用途

测试结束后，把分散的执行数据（用例结果、Bug 统计、性能数据）整理成一份规范报告。

## 使用场景

- 每轮测试结束
- 版本准出评审前
- 月度/季度质量汇报

## Prompt 本体

```markdown
根据以下测试执行数据，生成结构化测试报告。

## 项目信息
- 项目:
- 版本:
- 测试周期: [起-止]
- 测试负责人:

## 原始数据

### 用例执行数据
```
[粘贴用例执行结果，CSV/JSON/表格等]
```

### Bug 数据
```
[粘贴 Bug 列表：ID/标题/严重度/状态/模块]
```

### 性能测试数据（如有）
```
[接口/QPS/P99/错误率等]
```

### 环境信息
- 测试环境: [URL]
- 版本: [列表]

## 输出结构（参照 templates/testing/test-reports/test-report-template.md）

### 1. 摘要
[1-2 段核心结论]

### 2. 测试概述
- 测试范围
- 测试类型执行情况

### 3. 测试数据
- 用例执行统计
- Bug 统计（按严重度、按模块）
- Bug 趋势（发现/关闭/遗留）
- 性能测试结果

### 4. 风险评估
- 已知风险
- 回归风险

### 5. 质量评估
对照准出标准打分:
- 核心用例通过率
- 整体用例通过率
- P0/P1 遗留
- 性能达标
- 安全扫描

### 6. 准出意见
- ✅ 通过 / ⚠️ 有条件通过 / ❌ 不通过
- 理由

### 7. 上线建议
- 发布时间
- 灰度策略
- 监控重点

### 8. 改进建议
- 流程改进
- 对下个版本的建议

## 输出要求
- 图表用 Markdown 表格
- 数据要有对比（和基线/上版本）
- 结论要客观（数据说话）
- 风险要诚实（不粉饰）
```

## 使用示例

**输入**: 300 条用例的执行结果 + 42 个 Bug 数据

**输出**: 完整测试报告，可以直接作为准出报告
