# 日志聚类分析（开发场景）

## 用途

开发排查 Bug 时，面对大量日志找不到重点。让 AI 聚类 + 总结。

> 运维场景的日志分析见 [operations/log-analysis.md](../../operations/prompts/log-analysis.md)

## 使用场景

- 查某个接口的错误高峰期
- 排查偶发 Bug
- 验证某个新功能的日志是否正常

## Prompt 本体

```markdown
分析以下日志，找出异常模式。

## 日志时间范围
[开始] - [结束]

## 日志内容
```
[粘贴 50-500 行日志]
```

## 任务

### 1. 聚类错误
- 把相似的错误归类（如 DB 连接错误归为一类，而不是按具体 trace_id）
- 每类统计出现次数

### 2. 时间分析
- 标出错误**突增的时间点**
- 识别**周期性模式**
- 与正常时段对比

### 3. 关联分析
- 是否存在**错误链**（A 错 → B 错 → C 错）
- 是否存在**共同触发因素**

### 4. 优先级排序
按"严重度 × 频率"排序,给出前 5 个最值得关注的。

## 输出

### 错误聚类
| 类型 | 次数 | 首次 | 末次 | 严重度 |
|------|------|------|------|-------|

### 时间线
[关键事件的时间轴]

### 建议处理顺序
1. 立即处理: [列表 + 理由]
2. 尽快处理: [列表]
3. 可延后: [列表]

### 需要进一步调查
- 线索 1: [描述] → 建议查什么
- 线索 2: ...

### 需要补充的日志
（当前日志缺哪些信息，建议加）
- 加 X 字段
- 加 Y 时机的日志
```

## 使用示例

**输入**: 1000 行应用日志（混杂 INFO、WARN、ERROR）

**输出**: 6 类错误聚类 + 高峰期分析 + top 3 优先处理项
