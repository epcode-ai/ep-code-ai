# 错误排查

## 用途

看到陌生错误，让 AI 初步定位方向，节省搜索时间。

## 使用场景

- 遇到不懂的 error/exception
- 线上偶发 Bug 难复现
- 多人协作时别人的错误

```bash

```

### 3. 对应的修复方案

对每个原因,给出修复代码或思路。

### 4. 为避免类似问题建议的长期改进

```

## 使用示例

**输入**: `TypeError: Cannot read property 'map' of undefined` + 相关代码

**输出**:
1. 最可能: 接口返回的 data 字段为 null
2. 验证: 在 map 前 console.log
3. 修复: `(data || []).map(...)` 或加 null check
