# 技术方案决策

## 用途

在多个技术方案之间做选择时，让 AI 做结构化对比，输出可评审的 ADR。

## 使用场景

- 框架/库选型（React vs Vue）
- 数据库选型（MySQL vs PostgreSQL）
- 架构模式决策（单体 vs 微服务）
- 消息队列选型（Kafka vs RabbitMQ）

## Prompt 本体

```markdown
帮我在以下候选方案中做选择。

## 业务场景
[描述业务背景 + 关键约束]

## 我们的现状
- 团队规模: [人数 + 技术栈]
- 当前技术选型: [已有的技术]
- 运维能力: [DevOps 成熟度]
- 时间窗口: [决策时间 + 实施时间]
- 预算/成本约束: [如有]

## 候选方案

### 方案 A: [名称]
- 简述: ...
- 初步考虑的优缺点: ...

### 方案 B: [名称]
- 简述: ...
- 初步考虑的优缺点: ...

### 方案 C: [名称]（如有）
- ...

## 对比维度（按我们的优先级排序）
1. 性能
2. 实现复杂度
3. 维护成本
4. 可扩展性
5. 学习曲线
6. 社区活跃度
7. 风险
8. 长期演进

## 输出

### 1. 对比矩阵
| 维度 | 权重 | 方案 A | 方案 B | 方案 C |
|------|------|-------|-------|-------|
| 性能 | 高 | 评分 + 理由 | | |
| ... | | | | |

评分标准: 1-5 分（5 为最优）

### 2. 每个方案的决定性优缺点
不是简单列 5 条,而是找最关键的 1-2 条。

### 3. 综合推荐
**推荐方案 [X]**

理由:
1. ...
2. ...
3. ...

### 4. 反对意见预案
如果选方案 X，可能会有人提出：
- 反对点 1: [应对]
- 反对点 2: [应对]

### 5. 选定后的风险缓解
- 风险 1: [缓解措施]
- 风险 2: ...

### 6. ADR 格式输出
直接生成 [ADR 模板](../../../templates/development/adr-template.md) 格式的决策记录，
可以直接存档。
```

## 使用示例

**输入**: 微服务间通信选 gRPC 还是 REST + 我们的上下文

**输出**: 对比矩阵 + 推荐 gRPC + ADR 初稿
