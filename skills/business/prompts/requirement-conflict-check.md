# 需求冲突检测

## 用途

长文档 PRD 经常前后矛盾。AI 批量扫描比人眼更可靠。

## 使用场景

- PRD 定稿前
- 多人协作编辑的 PRD
- PRD 大版本更新后

## Prompt 本体

```markdown
你是一位资深产品分析师。扫描以下 PRD，找出**前后矛盾**或**逻辑冲突**。

## 检查维度

1. **术语一致性**：同一概念在不同章节定义是否一致
2. **规则一致性**：是否有"允许 X"和"禁止 X"并存
3. **数值一致性**：如第 2 章说 24h，第 5 章说 12h
4. **流程闭合性**：流程是否有"进了出不去"的状态
5. **权限一致性**：谁能做什么是否前后一致
6. **命名一致性**：同一字段名是否混用（如 userId / user_id / uid）
7. **状态与事件匹配**：状态机的每个转换都有对应事件描述

## 输出格式

每个冲突点输出:
| 冲突 # | 严重度 | 涉及章节 | 具体冲突描述 | 建议协调方案 |

## 严重度分级
- 🔴 High: 逻辑矛盾,必须修
- 🟡 Medium: 可能引起歧义,建议修
- 🟢 Low: 文字/风格问题,可选

## 附加输出
- 总冲突数
- 按类型分布
- 建议优先修复的 top 3

## PRD
[粘贴完整 PRD]
```

## 使用示例

**输入**: 一份 20 页的 PRD

**输出**:
```
| # | 严重度 | 章节 | 冲突 | 建议 |
|---|------|------|------|------|
| 1 | 🔴 | 3.2 vs 5.1 | 订单超时 3.2 说 30 分钟,5.1 说 1 小时 | 产品确认统一 |
| 2 | 🟡 | 2 vs 附录 | "会员" 在附录术语表说 VIP,正文直接用"VIP" | 统一术语 |

总冲突: 8 个（1 High + 4 Medium + 3 Low）
建议优先: # 1, # 3, # 7
```
