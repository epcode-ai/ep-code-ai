# 业务规则 → 决策表

## 用途

复杂业务规则用文字描述容易混乱。转成决策表后：
- 一目了然
- 可以直接翻译成代码
- 测试可以每行设计一个用例

## 使用场景

- PRD 里有复杂业务规则
- 需要和开发/测试对齐规则
- 做规则完整性检查

## Prompt 本体

```markdown
把下面的业务描述转成决策表。

## 要求
1. 列出所有**条件变量**（条件的维度）
2. 列出每个条件变量的**可选值**
3. 枚举**所有可能的组合**（笛卡尔积）
4. 对每个组合给出**预期结果**
5. 输出 Markdown 表格

## 特殊标注
- ⚠️ 原文未明确的组合（需要产品确认）
- ❌ 相互冲突的规则
- 🔄 可能需要递归的规则

## 输出结构

### 条件变量
| 变量 | 可选值 | 说明 |
|------|-------|------|

### 决策表
| # | 条件1 | 条件2 | 条件3 | 结果 | 备注 |
|---|-------|-------|-------|------|------|

### 发现
- 未明确的组合数: N
- 冲突: 无/列表
- 覆盖不全: 无/列表

## 业务描述
[粘贴自然语言的业务规则]
```

## 使用示例

**输入**: "订单金额 ≥ 100 普通用户享 5% 折扣，VIP 10%，新用户首单额外加 5%，但周末促销期间打 8 折不叠加..."

**输出**: 完整决策表,覆盖所有条件组合
