# 日志异常分析（运维场景）

## 用途

生产环境日志量大，运维排查需要快速聚类找规律。

> 开发场景见 [development/log-analysis](../../development/prompts/log-analysis.md)

## 使用场景

- 告警触发后快速扫描日志
- 疑似故障期的日志回溯
- 周期性质量审计

## Prompt 本体

```markdown
你是一位资深 SRE。分析以下生产日志，识别异常模式。

## 日志信息
- 服务: [服务名]
- 时间范围: [起-止]
- 日志量: [大概数量]
- 采样方式: 全量 / 随机采样 / 按 level 过滤

## 日志内容
```
[粘贴 100-2000 行日志，结构化或非结构化]
```

## 任务

### 1. 聚类（核心）
按"错误模式"聚类而不是按 trace_id 聚类:
- 忽略 ID/时间戳等易变字段
- 相似的错误归为一类
- 每类统计:
  - 次数
  - 首次/末次出现时间
  - 首次出现时是否有相关变更

### 2. 突增点识别
- 标出错误量**陡增**的时间点
- 标出**周期性**的错误
- 对照正常基线

### 3. 错误链分析
- 找出错误链（A 错 → B 错 → C 错）
- 识别级联错误的源头

### 4. 与已知故障的关联
- 如果错误模式与历史类似故障匹配,标注

### 5. 严重度评估
对每类错误判断:
- 🔴 严重: 需立即介入
- 🟡 中等: 观察
- 🟢 轻微: 可忽略

## 输出

### 异常模式 Top N
| # | 模式 | 次数 | 首现 | 严重度 | 可能原因 |

### 时间线
[关键事件的时间戳]

### 建议动作
1. 立即: [列表]
2. 短期: [列表]
3. 长期（改进建议）: [列表]

### 需要关注的指标
（基于日志推断监控缺口）
- 建议加监控的指标 1: ...
- 建议加告警的规则 1: ...

### 数据质量反馈
- 哪些字段缺失导致分析困难
- 建议增加什么字段（traceId、userId 等）
```

## 使用示例

**输入**: 告警期间的 10000 行日志

**输出**: 5 类错误聚类 + 突增点 + 推断根因 + 处置建议
