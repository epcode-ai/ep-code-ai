# 容量预测

## 用途

为扩容/缩容/大促准备做预测，避免"到用时才加机器"。

## 使用场景

- 季度容量规划
- 大促前预评估
- 成本优化（找浪费的资源）

## Prompt 本体

```markdown
基于以下历史数据,预测未来 [N 个月] 的容量需求。

## 历史数据

### 流量数据（最近 6-12 个月）
| 月份 | 日均 QPS | 峰值 QPS | DAU | 增长率 |
|------|---------|---------|-----|-------|
| | | | | |

### 资源数据
| 月份 | 平均 CPU | 平均内存 | 峰值 CPU | 存储（TB） |
|------|---------|---------|---------|----------|
| | | | | |

### 特殊事件
- [月份] [事件名] [流量倍数]
- [月份] [事件名] [流量倍数]

## 业务计划

- 未来 [N 个月] 预期用户增长: [X%]
- 即将上线的大功能: [列表 + 预估带来的流量增量]
- 即将到来的大促/活动: [时间 + 预估峰值倍数]
- 即将要拉新的地区/客群: [影响估计]

## 当前资源

- 服务实例数: [数字]
- 单实例规格: [CPU/内存]
- 数据库规格: [描述]
- 存储总量: [数字]
- CDN/带宽: [数字]
- 当前月成本: [金额]

## 约束

- 性能目标: P99 ≤ [X ms]
- 可用性目标: [99.9%]
- 预算上限: [金额/月]

## 输出

### 1. 预测结果

| 月份 | 预计日均 QPS | 预计峰值 | 预计 DAU | 说明 |
|------|------------|---------|---------|------|

预测方法:
- 基础: 历史线性/季节性趋势
- 调整: 业务计划驱动
- Buffer: [X]% 预留

### 2. 资源需求

| 资源 | 当前 | N 个月后 | 增量 | 扩容时机 |
|------|------|---------|------|---------|

### 3. 扩容建议
- **扩容方式**（水平/垂直）+ 理由
- **扩容节奏**（一次到位 vs 分批）
- **自动扩容规则**（HPA 设置建议）

### 4. 成本分析
- 预计新增成本: [金额]
- 成本优化机会: [列表]

### 5. 风险与应对
| 风险 | 应对 |
|------|------|

### 6. 需要进行的验证
- [ ] 压测验证到新 QPS 目标
- [ ] 演练扩容流程
- [ ] 准备降级方案

### 7. 反向检查
如果我的输入数据有异常（如某月数据缺失、明显离群点），请提醒。
```

## 使用示例

**输入**: 6 个月流量数据 + Q2 大促计划

**输出**: Q2 各月 QPS 预测 + 需扩容 3 次 + 成本增加 25% + 风险清单
