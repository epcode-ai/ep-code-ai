# 代码评审 Checklist 模板

> 参考规范：[docs/chapters/03-development/02-code-review.md](../../docs/chapters/03-development/02-code-review.md)
>
> **使用方式**：Reviewer 对照此清单评审 MR/PR，可以在 MR 评论里贴这份清单并逐项打勾。

---

## 评审基本信息

- **MR/PR**: [链接]
- **作者**: [@username]
- **Reviewer**: [@username]
- **评审时间**: YYYY-MM-DD HH:MM

## 一、通用检查（必做）

### 正确性
- [ ] 逻辑是否符合需求
- [ ] 边界条件处理完整（空、null、0、最大值、最小值）
- [ ] 异常路径有处理
- [ ] 并发/竞态条件考虑
- [ ] 数据类型正确（整数溢出、浮点精度、时区）

### 可读性
- [ ] 命名清晰（变量、函数、类）
- [ ] 函数不超过 50 行
- [ ] 嵌套不超过 3 层
- [ ] 复杂逻辑有注释（解释"为什么"）
- [ ] 魔法数字用常量替代

### 可维护性
- [ ] 无重复代码（DRY）
- [ ] 职责单一（SRP）
- [ ] 依赖方向合理（不循环）
- [ ] 配置与代码分离
- [ ] 日志级别使用得当

### 性能
- [ ] 无 N+1 查询
- [ ] 无循环里的网络调用
- [ ] 批量处理合理
- [ ] 缓存策略合理
- [ ] 资源及时释放

### 安全
- [ ] 输入校验（SQL 注入、XSS、CSRF）
- [ ] 敏感信息不进日志
- [ ] 密钥不硬编码
- [ ] 权限校验到位
- [ ] 错误信息不暴露内部细节

### 测试
- [ ] 新增代码有单元测试
- [ ] 测试覆盖正常流 + 异常流
- [ ] 测试命名清晰
- [ ] 无跳过的测试（或有合理注释）
- [ ] 覆盖率达标（≥ 80%）

### 文档
- [ ] API 文档已同步更新
- [ ] CHANGELOG 已更新
- [ ] 部署文档已更新（如需要）
- [ ] README 已更新（如需要）

---

## 二、按语言的特定检查

### TypeScript / JavaScript
- [ ] 用 `const`/`let`，禁用 `var`
- [ ] 没有 `any`（或有合理理由）
- [ ] Promise 有 `.catch` 或 try-catch
- [ ] 异步函数不吞异常
- [ ] null/undefined 区分清楚

### Python
- [ ] 类型注解完整
- [ ] 异常类型具体
- [ ] 资源用 `with` 管理
- [ ] 字符串格式化用 f-string
- [ ] 无可变默认参数陷阱

### Java / Kotlin
- [ ] null 处理（Optional / @Nullable）
- [ ] 资源关闭（try-with-resources）
- [ ] 线程安全
- [ ] equals/hashCode 同步

### Go
- [ ] error 都有处理
- [ ] goroutine 有退出机制
- [ ] channel 避免泄漏
- [ ] context 正确传递

### SQL
- [ ] 索引命中（EXPLAIN 过）
- [ ] 大表更新有 WHERE
- [ ] 避免 `SELECT *`
- [ ] 事务范围合理

### Swift（针对 app/ 目录）
- [ ] Optional 正确解包
- [ ] weak / strong 引用合理
- [ ] @MainActor 使用正确
- [ ] 内存管理（避免循环引用）

## 三、评审意见分级

在评论时使用标签：

| 标签 | 含义 |
|------|------|
| `must-fix` / `🔴` | 必须修改才能合入 |
| `should-fix` / `🟡` | 建议修改 |
| `nit` / `🟢` | 细节建议，可选 |
| `question` / `❓` | 提问 |
| `praise` / `🎉` | 写得好 |

## 四、评审结论

- [ ] ✅ Approve（通过）
- [ ] 🔄 Request Changes（需改动后再审）
- [ ] 💬 Comment Only（只留评论，不投票）

**综合意见**：
[1-2 段话总结]

**主要问题**（如有 Request Changes）：
1. ...
2. ...

**亮点**（如有）：
- ...

## 五、作者响应模板（给作者用）

对每条评审意见，回复格式：

```
@reviewer 

对于 [xxx] 这条意见：
- [ ] 已修改（commit: abc123）
- [ ] 已讨论，保留现状（理由：...）
- [ ] 待下个 MR 处理（新 Issue: #xxx）
```
